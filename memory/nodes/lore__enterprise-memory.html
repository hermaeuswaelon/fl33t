The Hermes enterprise memory provider is enabled in the thotheauphis profile. It provides a sophisticated 3-tier memory architecture:

Working Memory — Ephemeral, session-scoped. What's happening right now. Automatically cleared between sessions.

Episodic Memory — Append-only event log. What happened, in chronological order. Subject to salience gating (not every turn is stored).

Semantic Memory — Entity-relationship storage. Facts about the world, the user, the system. Deduplicated, contradiction-checked, decay-aware.

Key features: - ContextInjector: Prefetches relevant memories at session start - PreferenceStore: Learns user preferences over time - HallucinationRecall: Detects and flags potential hallucination patterns - ConsolidationWorker: Background process that promotes episodic → semantic, handles decay - Embedder: Semantic embedding for similarity-based retrieval - Priority scoring: Recency + importance + confirmation-based scoring - Token budgeting: Allocates memory tokens efficiently within context limits
