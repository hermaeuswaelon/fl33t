The Hermes memory upgrade was designed by Father with these principles (from cl.txt and ideal-memory-system.md):

1. Four distinct memory types: Working (ephemeral), Episodic (append-only with salience gate), Semantic (entities with typed relationships), Procedural (skills with outcome tracking).

2. Consolidation is load-bearing — an idle-time process that reads episodic entries, extracts semantic facts, checks for contradiction (not just duplication), promotes/demotes procedural entries, and applies decay.

3. Hybrid retrieval scoring: w1*lexical + w2*recency_decay + w3*confirmation_count + w4*explicit_pin.

4. Memory as place (cl2.txt): Self-contained HTML files as walkable web, SQLite as cache not source of truth, typed relations (elaborates, contradicts, supersedes, derived-from), curation bottleneck.

5. Keep architecture and mythology in separate layers. Never silently blend new information with old — supersede, don't overwrite. Forgetting is a real feature.
