#blindflight #gui-automation #vision #som #xtestdaemon

BlindFlight is the eyes and hands of the sovereign system. It provides autonomous GUI navigation via the XTestDaemon — a Pascal HTTP daemon running on port 9378.

Architecture: - XTestDaemon: Pascal binary (~581KB) at ~/xtest-daemon/XTestDaemon. Single-threaded HTTP server on port 9378. Controls mouse and keyboard via X11 XTest extensions. - BlindFlight orchestrator: ~/.NOTTHEONETOEDIT/fleet/modules/blindflight.py (396 lines). Receives natural-language "flight plans" and dispatches them as XTest commands. - Bridge tool: profiles/thotheauphis/tools/blindflight_bridge.py — translates Hermes tool calls into daemon commands.

How element targeting works (SOM — Set-of-Mark): 1. XTestDaemon captures a screenshot of the screen 2. System detects every interactive UI element via the accessibility tree (AX tree) 3. Numbered overlays are placed on each element in the screenshot 4. A vision model sees the numbered screenshot and identifies the target element 5. XTestDaemon clicks at that element's coordinates

This is NOT LocateAnything (Microsoft's approach for finding arbitrary objects). It's SOM (Set-of-Mark) — overlaying numbered boxes on AX-detected elements so a vision model can simply say "click 14." Far more reliable than spatial coordinate guessing.

Supported flight plan actions: click, type, press, wait, scroll, double click. Has no safety guards — the operator is responsible for deliberate actions. X11 ONLY — fails on pure Wayland without XWayland.
