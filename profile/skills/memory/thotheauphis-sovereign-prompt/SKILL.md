## System Maintenance & Optimization

When invoked with `/thotheauphis-sovereign-prompt optimize`, run a **fast parallel 7-phase system sweep**. Core principle: **act fast** — batch independent operations, delegate deep analysis to subagents, do not over-analyze before acting.

### Procedure (7 Phases — run together where possible)

**Phase 1 — Executors:** Check thoth_recursor, thoth_daemon, ares_witness, hermes_gateway daemons and their IPC sockets.

**Phase 2 — Hardware Tune:** Set CPU governor=performance on all cores. Tune VM: swappiness=10, vfs_cache_pressure=50, dirty_ratio=30. Drop stale caches.

**Phase 3 — Storage Cleanup:** Remove build artifacts (warp/target/ is 20G+), purge pip cache, apt-get clean+autoremove, journalctl vacuum 3d, empty trash.

**Phase 4 — USB Backup:** Find USB, fix root ownership (`sudo chown $USER:$USER /run/media/$USER/*`), tar profile tarball (fast, atomic), background rsync for large dirs.

**Phase 5 — Script/Cron Audit:** Fix missing shebangs, `~/nottheonetoedit`→`~/.NOTTHEONETOEDIT` path bugs, ZODB dependency in sms-stats (rewrite to use sms CLI).

**Phase 6 — Identity Integrity:** Run `identity-integrity-check.sh` — all 6 layers must pass with 0 errors.

**Phase 7 — Parallel Testing:** Delegate subagents for deep executor/fleet/SMS testing. Run hardware tests in parallel (GPU, NVMe speed, network, Docker).

### Pitfalls
- **"wait less, do more"** — user prefers fast parallel action over analysis. Run fixes simultaneously. If a tool has a 600s timeout, use background mode for long transfers.
- **USB owned by root** — always `sudo chown` before USB writes.
- **rsync over USB** — background mode required for files over 5GB. Use tar for critical profile data.
- **SMS scripts need sms CLI** — scripts that import ZODB directly fail from cron (system python lacks the venv). Rewrite to call `sms status` and parse JSON from stdout.
- **sudo auth** — expires after one call. Batch sudo commands in the same terminal() call.
- **warp/target/** — safe to delete entirely (20G Rust build artifacts). Source is in `crates/` and `Cargo.lock`.
