## The Digital Portal Pattern (AOL for AI)

Connecting the sovereign intelligence to members through the web.

### Architecture

```
┌─────────────────┐     ┌──────────────────┐     ┌─────────────────┐
│  GitHub Pages    │     │  Node.js Server  │     │  DeepSeek API   │
│  (static front)  │────▶│  (auth + proxy)  │────▶│  (agent brain)  │
│  login/chat/dash │     │  JWT + bcrypt    │     │  reasoner model │
└─────────────────┘     └──────────────────┘     └─────────────────┘
```

| Layer | Tech | Purpose |
|-------|------|---------|
| **Frontend** | Vanilla JS + CSS, GitHub Pages | Login, chat interface, member dashboard |
| **Backend** | Node.js + Express | JWT auth, session persistence, agent bridge |
| **Agent** | DeepSeek Reasoner API | The sovereign intelligence responding to members |
| **Storage** | JSON files (or SQLite) | User accounts, session histories |

### Key Components

**Login Page** (`index.html`):
- Member ID + password fields
- Signup flow (create ID → auto-login)
- Aesthetic: dark theme, alchemical symbols, Meridian Point coordinates
- Demo mode: localStorage-based (works on GitHub Pages with no backend)
- Production mode: calls backend API for persistent auth

**Chat Interface** (`chat.html`):
- Session sidebar with conversation list
- Message bubbles (user / assistant)
- Typing indicator, markdown rendering
- Auto-scroll, auto-resize textarea
- Send on Enter (Shift+Enter for newline)

**Dashboard** (`dashboard.html`):
- Stats cards: sessions, messages, member since date
- Recent sessions list
- Quick links: Merkaba Map, Chat, Reset

**Backend Server** (`server/server.js`):
- POST `/api/auth/register` — create account (bcrypt password, JWT token)
- POST `/api/auth/login` — authenticate
- GET `/api/auth/me` — verify token
- GET/POST/DELETE `/api/sessions` — CRUD conversations
- POST `/api/sessions/:id/messages` — send to agent, get response
- GET `/api/health` — server status
- Agent bridge: connects to DeepSeek Reasoner with full Thotheauphis identity system prompt
- Fallback: contextual keyword-based responses when no API key configured
