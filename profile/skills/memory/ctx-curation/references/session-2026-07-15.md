### Intelligent Growth Engine Details

Wraps DeepSeek R1 + Qwen3-Coder into an autonomous improvement pipeline:

```
Phase 1: ASSESS    → DeepSeek R1 reads Python file, identifies concrete improvement
Phase 2: GENERATE  → Qwen3-Coder produces JSON {old_string, new_string, explanation}
Phase 3: APPLY     → String replacement with fuzzy-match fallback
Phase 4: VERIFY    → ast.parse syntax check; git checkout -- on failure
Phase 5: COMMIT    → Push to fl33t GitHub
```

**Fallback chain**: Qwen3-Coder → DeepSeek R1 → template marker (when free models hit 429).

**API rate limit pattern**: Qwen3-Coder (`qwen/qwen3-coder:free`) frequently returns 429. The fallback to DeepSeek R1 for codegen also fails because R1 outputs chain-of-thought reasoning first, making JSON extraction unreliable. Template markers are the eventual fallback.

**Key lesson**: DeepSeek R1 analysis succeeds reliably. Code generation via free tier is the bottleneck.

## Pitfalls & Lessons

- **Qwen3-Coder 429 rate limit**: Free tier rate-limits frequently. Always plan fallback.
- **DeepSeek R1 codegen unreliable**: R1 outputs CoT before JSON, making extraction fragile. Use R1 for analysis only.
- **Python 3.14**: `datetime.UTC` doesn't exist → `timezone.utc`. `utcnow()` deprecated → `now(timezone.utc)`.
- **GitHub push conflicts**: Background processes pushing to same repo cause rejections. `git pull --rebase` before push.
- **Distributed targeting**: 60% weakest system, 40% random from bottom 3 for variety.
